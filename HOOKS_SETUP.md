# Автоматическая проверка технических текстов через Hooks

Эта система автоматически проверяет и анализирует все markdown файлы, которые создаёт Claude Code, используя агента tech-writing на основе методологии **«Пиши, сокращай»** Максима Ильяхова.

## Как это работает

Когда вы создаёте или изменяете markdown файл (`.md`) в Claude Code, срабатывает hook, который:

1. **Логирует** все проверенные файлы
2. **Анализирует** текст на предмет:
   - Стоп-слов (безусловно, разумеется, как правило)
   - Неопределённости (некоторый, определённый, целый ряд)
   - Заумности (осуществлять, использовать)
   - Многословия и лишних фраз
   - Канцелярита и штампов
   - Неясных формулировок
3. **Даёт рекомендации** по улучшению текста с конкретными примерами

## Установка

### Способ 1: Проектные hooks (только для этого проекта)

Файлы уже созданы в репозитории:
- `.claude/settings.json` — конфигурация hooks
- `.claude/hooks/check-tech-writing.sh` — скрипт для логирования

Hooks будут работать автоматически при работе с этим проектом.

### Способ 2: Глобальные hooks (для всех проектов)

Если вы хотите использовать автоматическую проверку во всех проектах:

1. Скопируйте конфигурацию в домашнюю директорию:

```bash
cp .claude/settings.json ~/.claude/settings.json
```

2. Или отредактируйте `~/.claude/settings.json` и добавьте такую же конфигурацию.

> **Важно:** Если в `~/.claude/settings.json` уже есть другие настройки, **не перезаписывайте** файл целиком. Просто добавьте секцию `hooks` к существующей конфигурации.

## Как использовать

### Автоматическая проверка

Просто работайте как обычно. При создании или редактировании файла `.md` Claude автоматически:

1. Сохранит событие в лог (`.claude/tech-writing-checks.log`)
2. Анализирует текст и даст вам обратную связь

Пример обратной связи:

```
✓ Текст в порядке

ИЛИ

⚠️ Найдены проблемы:

1. **Стоп-слова** (строка 5):
   ❌ "Безусловно, мы должны пересмотреть..."
   ✅ "Мы должны пересмотреть..."

2. **Многословие** (строка 12):
   ❌ "целый ряд известных компаний"
   ✅ "Google, Facebook, Microsoft"

3. **Неясность** (строка 20):
   Добавьте конкретные примеры вместо общих формулировок.
```

### Явная проверка файла

Если хотите явно попросить проверку:

```bash
# С использованием alias
tech-writer "Проверь файл README.md на стиль"

# Или полная команда
claude --include ~/.claude/agents/tech-writing-agent/tech-writing-agent.md "Проверь файл README.md на стиль"
```

## Логирование

Все проверки сохраняются в файл `.claude/tech-writing-checks.log`:

```bash
cat .claude/tech-writing-checks.log
```

Вывод:
```
[2024-12-13 15:00:23] Проверка: README.md
[2024-12-13 15:02:15] Проверка: INSTALLATION.md
[2024-12-13 15:05:42] Проверка: HOOKS_SETUP.md
```

## Настройка

### Отключение hooks

Если нужно временно отключить автоматическую проверку, отредактируйте `.claude/settings.json`:

```json
{
  "hooks": {
    "PostToolUse": []  // Пустой массив отключает hooks
  }
}
```

### Фильтр по расширению

Hooks срабатывают только на файлы с расширением `.md`. Если нужно добавить другие расширения (`.mdx`, `.rst`), отредактируйте скрипт в `.claude/hooks/check-tech-writing.sh`:

```bash
[[ ! "$FILE_PATH" =~ \.(md|mdx|rst)$ ]]  # Добавьте нужные расширения
```

### Минимальный размер файла

По умолчанию не проверяются файлы меньше 100 байт. Измените это в скрипте:

```bash
if [ "$FILE_SIZE" -lt 500 ]; then  # Минимум 500 байт
```

## Отключение для конкретного файла

Если файл не нужно проверять, добавьте комментарий в начало:

```markdown
<!-- no-tech-writing-check -->

# Ваш контент...
```

Затем отредактируйте скрипт для проверки этого комментария (опционально).

## Требования

- **Claude Code** установлен и работает
- **.md файлы** находятся в проекте
- **Интернет подключение** для работы с Claude API
- Конфигурация hooks в `.claude/settings.json` или `~/.claude/settings.json`

## Встроение в CI/CD (опционально)

Если нужно проверять файлы в CI/CD pipeline:

```bash
#!/bin/bash
for file in $(git diff --name-only | grep '\.md$'); do
  claude --include ~/.claude/agents/tech-writing-agent/tech-writing-agent.md "Проверь файл $file"
done
```

## Примеры использования

### Пример 1: Создание нового README

Когда вы создаёте `README.md`, Claude автоматически проверит его:

```bash
claude "Создай README.md для проекта tech-writing-agent"
```

Вывод:
```
ℹ️  [Проверка текста]
Анализ README.md...

✓ Структура хорошая
⚠️  Найдены 2 проблемы:
1. Уберите "как правило" в описании
2. Добавьте больше конкретики в примеры
```

### Пример 2: Редактирование документации

При редактировании файла:

```bash
claude "Добавь раздел 'Установка' в DOCUMENTATION.md"
```

Hook автоматически проверит добавленный текст.

### Пример 3: Применение исправлений

Если Claude предложил исправления:

```bash
claude --include ~/.claude/agents/tech-writing-agent/tech-writing-agent.md "Примени эти правки в файл INDEX.md: ..."
```

## Отладка

Если hooks не работают:

1. **Проверьте конфигурацию:**
   ```bash
   cat .claude/settings.json
   ```

2. **Убедитесь, что скрипт исполняемый:**
   ```bash
   ls -la .claude/hooks/check-tech-writing.sh
   # Должны быть права -rwx-
   ```

3. **Проверьте лог:**
   ```bash
   cat .claude/tech-writing-checks.log
   ```

4. **Запустите с флагом отладки:**
   ```bash
   claude --debug
   ```

## Стоимость и производительность

- Рекомендуется использовать для важных документов
- Для частых проверок можно отключить hook и проверять вручную по необходимости
