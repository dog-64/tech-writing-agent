# Очистка и финальное состояние скриптов

## Что произошло

### ❌ Удалён: `check-tech-writing.sh`

**Причина:** Скрипт был неработающим прототипом, который только логировал события, но не выполнял анализ.

```bash
# Что он делал (бесполезно):
echo "[2024-12-13 15:00:23] Проверка: README.md" >> .claude/tech-writing-checks.log
```

### ✅ Остался: `analyze-markdown.sh`

**Статус:** Единственный активный скрипт, используется в hooks.

```bash
# Что он делает (полезно):
- Парсит JSON из stdin
- Анализирует текст
- Ищет стоп-слова и заумные слова
- Выводит рекомендации
- Показывает статистику
```

## Как работает текущая конфигурация

```
claude "Создай README.md"
         ↓
Write инструмент создаёт файл
         ↓
PostToolUse event срабатывает
         ↓
Запускается: ~/.claude/hooks/analyze-markdown.sh
         ↓
Скрипт получает JSON через stdin с содержимым файла
         ↓
Анализирует: стоп-слова, заумные слова, статистика
         ↓
Выводит результат в stdout
         ↓
Claude видит результат и показывает пользователю
         ↓
Пользователь видит рекомендации и решает применять ли их
```

## Что удалить из .claude/

Если вы хотите полностью очистить директорию hooks, вы можете удалить `check-tech-writing.sh` глобально (он там скопирован):

```bash
rm ~/.claude/hooks/check-tech-writing.sh  # Если существует
```

Но это уже сделано в проектной директории.

## Финальная конфигурация

**`.claude/settings.json` (проект):**
```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "if command -v jq &> /dev/null; then \"$CLAUDE_PROJECT_DIR\"/.claude/hooks/analyze-markdown.sh; fi"
      }]
    }]
  }
}
```

**`~/.claude/settings.json` (глобально):**
Идентично - work на всех проектах.
