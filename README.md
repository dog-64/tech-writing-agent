# Tech Writing Agent 🤖

Агент для автоматической проверки и улучшения технических текстов по методологии **«Пиши, сокращай»** Максима Ильяхова.

Агент анализирует markdown файлы на стоп-слова, многословие, неясность и канцелярит, помогая писать краткие, ясные и убедительные тексты.

## Быстрый старт

```bash
# 1. Установить агент и настроить hooks
bash setup.sh

# 2. Начать использовать
claude "Создай README.md для моего проекта"
# ↓ Автоматически проверится на качество текста
```

## Что это дает?

✅ Автоматическая проверка всех `.md` файлов при создании/редактировании
✅ Анализ по методологии Ильяхова: стоп-слова, заумность, многословие
✅ Конкретные рекомендации с примерами
✅ Работает во всех проектах (глобальная настройка)

## 🚀 Установка

### Шаг 1: Запустить setup.sh

```bash
cd ~/Sites/tech-writing-agent
bash setup.sh
```

Скрипт автоматически:
- ✅ Установит агент в `~/.claude/agents/tech-writing-agent/`
- ✅ Настроит глобальные hooks в `~/.claude/settings.json`
- ✅ Позволит проверять все `.md` файлы во всех проектах

### Шаг 2: Протестировать

```bash
# В любом проекте создайте тестовый файл:
claude "Создай файл TEST.md с текстом: 'Безусловно, как правило, мы должны использовать различные подходы'"

# Вы должны увидеть анализ:
# ⚠️  СТОП-СЛОВА:
#   • 'безусловно' - удаляйте
#   • 'как правило' - удаляйте
```

## Как пользоваться

### Автоматическая проверка (рекомендуется)

```bash

claude "Создай DOCUMENTATION.md для моего API"
# ↓ Проверка срабатывает сама
```

### Явная проверка конкретного файла

```bash

# Или полная команда
claude --include ~/.claude/agents/tech-writing-agent/tech-writing-agent.md "Проверь файл README.md"
```

### Добавить alias для удобства

```bash

# Затем используйте:
tech-writer "Проверь этот документ на стоп-слова"
```

## Что проверяется

### 1️⃣ Стоп-слова
Слова и фразы, которые удаляются без потери смысла:
- Вводные конструкции: безусловно, разумеется, как правило
- Неопределённость: некоторый, определённый, целый ряд
- Размытость: довольно, достаточно, весьма, практически

### 2️⃣ Заумные слова
Замена сложных слов простыми:
- осуществлять → делать
- использовать → применять, брать
- реализовать → сделать, создать
- функционировать → работать

### 3️⃣ Многословие
Удаление лишних слов и фраз без потери смысла

### 4️⃣ Канцелярит и штампы
Избавление от формальных и пустых фраз

### 5️⃣ Неясность
Предложение конкретизировать примерами

## Глобальные vs Проектные hooks

| Настройка | Где работает | Где находится | Когда использовать |
|-----------|-------------|---------------|------------------|
| **Глобальные** | Все проекты | `~/.claude/settings.json` | Вы хотите проверку везде |
| **Проектные** | Текущий проект | `.claude/settings.json` | Специальная настройка |

По умолчанию `setup.sh` настраивает **глобальные** hooks.

## FAQ

**Q: Может ли hook вызвать агента напрямую?**
A: Нет, но результаты анализа помогут Claude дать рекомендации.

**Q: Проверяются ли другие форматы (`.txt`, `.py`)?**
A: Нет, только `.md` файлы.

**Q: Может ли hook автоматически исправить файл?**
A: Нет, но Claude может предложить исправления, которые вы примените.

**Q: Как добавить новые стоп-слова?**
A: Отредактируйте `.claude/hooks/analyze-markdown.sh` и добавьте слова в массив `STOPWORDS`.

**Q: Почему hook не срабатывает?**
A: Проверьте:
- Установлена ли jq: `jq --version`
- Есть ли скрипт: `ls ~/.claude/hooks/analyze-markdown.sh`
- Создаёте ли вы `.md` файлы (другие форматы игнорируются)
