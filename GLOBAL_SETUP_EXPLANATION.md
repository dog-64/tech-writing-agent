# Глобальная настройка Tech Writing Agent

## Что произошло

Когда вы запустили `bash setup.sh`, произошло следующее:

### 1. Установка агента
- Файл `tech-writing-agent.md` скопирован в `~/.claude/agents/tech-writing-agent/`
- Этот путь используется Claude Code для доступа к агенту

### 2. Настройка глобальных hooks
- Скрипт проверил существование `~/.claude/settings.json`
- Если файл не существовал - создал его с конфигурацией hooks
- Если файл существовал - объединил конфигурацию используя `jq`:
  - Старые параметры остались нетронутыми
  - Добавилась новая секция `"hooks"`

### 3. Как выглядит результат

```json
{
  "statusLine": { ... },           // ← Ваша старая конфигурация
  "alwaysThinkingEnabled": true,   // ← осталась нетронутой
  "model": "haiku",                // ←
  "hooks": {                        // ← НОВОЕ: автоматическая проверка текстов
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "prompt",
            "prompt": "Проверить markdown файл на стиль..."
          }
        ]
      }
    ]
  }
}
```

## Как это работает

### Событие срабатывает

Когда вы создаёте или редактируете файл, Claude Code запускает event **PostToolUse** (после выполнения Write или Edit инструмента).

```
claude "Создай README.md"
    ↓
Write инструмент создаёт файл
    ↓
🔔 PostToolUse event срабатывает
    ↓
Проверяется matcher "Write|Edit" → ✅ совпадает
    ↓
Выполняется hook type: "prompt"
    ↓
Claude анализирует файл на стиль
    ↓
Выдаёт рекомендации по улучшению
```

### Фильтрация

Hook имеет встроенную логику:
- Проверяет только `.md` файлы (остальные игнорируются)
- Если файл отличного качества → просто говорит "OK"
- Если есть проблемы → даёт конкретные рекомендации с примерами

## Где это работает

✅ **Работает везде:**
- Все проекты на вашей машине
- Все созданные/отредактированные `.md` файлы
- Как в Claude Code, так и в других контекстах

❌ **НЕ работает:**
- Файлы, которые не являются markdown (`.py`, `.js`, `.txt` и т.д.)
- Файлы меньше 100 байт (конфигурируется в скрипте)

## Уровни конфигурации

### 1. Глобальные hooks (машина) 🌍
📁 `~/.claude/settings.json`
- Работают для ВСЕх проектов
- Применяются ко ВСЕМ файлам

### 2. Проектные hooks (текущий проект) 📂
📁 `.claude/settings.json` (в корне проекта)
- Работают только для этого проекта
- Переопределяют глобальные настройки

### 3. Приоритет
```
Проектные hooks (.claude/settings.json)
        ↓ если не найдены
Глобальные hooks (~/.claude/settings.json)
        ↓ если ничего нет
Используются defaults Claude Code
```

## Как отключить

### Опция 1: Отключить для всех проектов
```bash
# Удалите или закомментируйте секцию "hooks" в ~/.claude/settings.json
nano ~/.claude/settings.json
```

### Опция 2: Отключить для одного проекта
```bash
# Создайте .claude/settings.json в корне проекта с пустыми hooks
echo '{"hooks": {"PostToolUse": []}}' > .claude/settings.json
```

### Опция 3: Просто игнорировать рекомендации
- Claude выдаст рекомендации, но вы можете их не применять
- Проверка не блокирует вашу работу

## Как переустановить

Если что-то пошло не так:

```bash
# 1. Восстановить backup (если есть)
cp ~/.claude/settings.json.backup ~/.claude/settings.json

# 2. Запустить setup.sh заново
bash setup.sh

# 3. Или вручную добавить конфигурацию в ~/.claude/settings.json
```

## Когда срабатывает hook

### Срабатывает 🟢
```bash
claude "Создай файл DOCUMENTATION.md"     # ← Write инструмент
claude "Исправь опечатку в README.md"     # ← Edit инструмент
```

### НЕ срабатывает 🔴
```bash
claude "Прочитай файл README.md"          # ← Read инструмент (не ловится)
claude "Выполни команду ls"               # ← Bash инструмент (не ловится)
claude "Что в файле index.js?"            # ← Grep инструмент (не ловится)
```

## Отладка

Если hooks не работают:

```bash
# 1. Проверьте что hooks есть в settings.json
cat ~/.claude/settings.json | jq '.hooks'

# 2. Убедитесь что jq установлен
jq --version

# 3. Запустите Claude с флагом отладки
claude --debug

# 4. Посмотрите логи
cat ~/.claude/debug/*.log | tail -100
```

## Стоимость

⚠️ **Важно:** Каждая проверка использует Claude API
- Каждый созданный/отредактированный `.md` файл = 1 API call
- Это может увеличить количество запросов к API
- Если это критично - отключите hooks или используйте проектные настройки

## Примеры

### Пример 1: Создание документации

```bash
claude "Создай INSTALLATION.md с инструкциями по установке"
```

**Что происходит:**
1. Claude создаёт файл INSTALLATION.md
2. Hook срабатывает
3. Claude анализирует файл на стиль
4. Выдаёт рекомендации (или говорит что всё ОК)

### Пример 2: Редактирование документа

```bash
claude "Добавь раздел 'API Reference' в README.md"
```

**Что происходит:**
1. Claude редактирует README.md
2. Hook срабатывает (Edit инструмент)
3. Claude проверяет добавленный текст
4. Выдаёт рекомендации по улучшению

### Пример 3: Отключение для проекта

```bash
# В корне проекта
echo '{"hooks": {"PostToolUse": []}}' > .claude/settings.json

# Теперь для этого проекта hooks отключены
claude "Создай README.md"  # ← Проверка не срабатывает
```
