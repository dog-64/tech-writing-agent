# Быстрая настройка автоматической проверки текстов

## За 1 минуту 🚀

### Шаг 1: Запустите setup.sh

```bash
cd ~/Sites/tech-writing-agent
bash setup.sh
```

**Всё!** Скрипт автоматически:

- ✅ Установит агент в `~/.claude/agents/tech-writing-agent/`
- ✅ Добавит глобальные hooks в `~/.claude/settings.json`
- ✅ Включит автоматическую проверку для ВСЕХ проектов

### Шаг 2: Протестируйте

В любом проекте создайте тестовый файл:

```bash
claude "Создай файл TEST.md с текстом: 'Безусловно, как правило, мы должны использовать различные подходы для автоматизации целого ряда процессов'"
```

Вы должны увидеть предложение исправить текст 👇

## Что происходит автоматически

Каждый раз когда вы пишете/редактируете `.md` файл **в любом проекте**:

1. Claude автоматически читает файл
2. Проверяет его на стоп-слова, многословие, неясность
3. Даёт конкретные рекомендации с примерами
4. Файл останется нетронутым - вы сам решаете применять ли исправления

## Глобальные vs Проектные hooks

| Вариант                        | Где работает          | Где настраивается         | Когда использовать                |
|--------------------------------|-----------------------|---------------------------|-----------------------------------|
| **Глобальные** (рекомендуется) | ВСЕ проекты           | `~/.claude/settings.json` | Вы хотите проверку везде          |
| Проектные                      | Только текущий проект | `.claude/settings.json`   | Специальная настройка для проекта |

setup.sh автоматически настраивает **глобальные** hooks, так что вам больше ничего не нужно!

## Отключение (если нужно)

Если нужно временно отключить проверку:

```bash
# Отредактируйте ~/.claude/settings.json и удалите секцию "hooks"
nano ~/.claude/settings.json
```

Или просто не обращайте внимание на рекомендации Claude - вы всегда можете их проигнорировать.

## Дополнительные команды

**Явная проверка конкретного файла:**

```bash
tech-writer "Проверь файл README.md на стиль"
```

**Проверка несколько файлов:**

```bash
claude --include ~/.claude/agents/tech-writing-agent/tech-writing-agent.md "Проверь все файлы в папке docs/"
```
